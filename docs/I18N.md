# Internationalization (i18n)

**Current state (2026):** The app ships **en, de, fr, it** UI strings under `frontend/src/i18n/locales/`. The guided tour uses `frontend/src/i18n/tourStepBundles.ts` plus `tourDe.ts` / `tourIt.ts` (all four locales have full step copy). After key changes, run `cd frontend && npm run check-locale-keys` and the Vitest i18n tests.

When adding or changing copy, prefer these patterns:

1. **Centralize strings** — Prefer a single module or JSON map per locale for user-visible text (buttons, errors, Settings labels), rather than scattering literals in JSX.
2. **Avoid string concatenation** for sentences — Use templates or ICU-style placeholders for pluralization and word order.
3. **Locale-aware formatting** — Dates and numbers already benefit from `Intl` when you add locale detection.
4. **Electron** — `app.getLocale()` can drive the initial UI language; persist choice in app settings.

This is a gradual migration: new features should avoid hard-coding long strings inline when a shared copy module already exists for that screen.
